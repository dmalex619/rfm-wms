Подтверждение прихода

 

При подтверждении выполнения прихода система производит следующие операции:

поступившие товары, в соответствии с их состоянием, принимаются на остатки;

для каждого контейнера выполняется операция подбора ячейки (по правилам подбора): 

для контейнера с годным подбирается подходящая по габаритам и грузоподъемности ячейка хранения, по возможности ближайшая к ячейке пикинга товара,

для контейнера с товаром в другом состоянии (брак) подбирается ячейка, закрепленная за этим состоянием;

если ячейка подобрана, формируется операция – задание на транспортировку контейнера в эту ячейку,

если ячейка не подобрана, контейнер остается в том же состоянии, и для его размещения в ячейке сотруднику нужно будет либо повторно выполнить попытку размещения всех контейнеров прихода, либо вручную сформировать операцию транспортировки этого контейнера в самостоятельно выбранную ячейку.

Для подтвержденного прихода невозможны операции редактирования или удаления.

Выполнять подтверждение следует только после того, как будет внесены сведения обо всех поступивших в данном приходе контейнерах.

 

Если часть данных о поступивших товарах / контейнерах не была введена, а подтверждение уже выполнено, товар в этих контейнерах будет считаться неполученным и не поступит на учет в систему. Для исправления такой ситуации придется вводить новый (корректирующий) приход либо выполнять операцию "исправление состояния контейнера".
