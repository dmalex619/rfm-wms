Список операций транспортировки

 

Экранная форма для отображения списка операций транспортировки состоит из двух страниц: "Условия" и "Операции".

 

На первой странице "Условия" можно назначить условия, которые ограничат набор выводимых данных:

дата задания (диапазон, от … до …),

ячейка-источник (откуда):

щтрих-код (контекст),

адрес (контекст),

складская зона (отметка одного или нескольких значений в списке),

тип складской зоны (отметка одного или нескольких значений в списке),

ячейка-приемник (куда):

щтрих-код (контекст),

адрес (контекст),

складская зона (отметка одного или нескольких значений в списке),

тип складской зоны (отметка одного или нескольких значений в списке),

контейнер:

штрих-код,

код (один или список кодов через запятую),

пользователь, выполнявший операцию (отметка одного или нескольких значений в списке),

мобильное устройство, использованное при выполнении операции (отметка одного или нескольких значений в списке),

состояние операций (выбор одного варианта: любое / не выполнены / выполнены успешно / выполнены с ошибкой).

На странице "Операции" будут отображены только те операции транспортировки, которые удовлетворяют назначенным условиям.

 

Для назначения условий на операции транспортировки используются кнопки:

Очистка всех условий

 

На второй странице "Операции" показаны данные об операциях транспортировки.

Одна строка таблицы соответствует одной операции (или этапу операции, если операция выполнялась с использованием буферных ячеек):

уникальный код операции транспортировки (ID),

код операции в учетной системе (ERPCode),

приоритет операции,

код контейнера,

штрих-код контейнера,

владелец контейнера (может быть пуст, т.н. "общий товар"),

состояние товара,

ячейка-источник:

адрес ячейки,

штрих-код ячейки,

адрес буферной ячейки,

складская зона,

тип зоны,

ячейка-приемник:

адрес ячейки,

штрих-код ячейки,

адрес буферной ячейки,

складская зона,

тип зоны,

дата-время формирования операции,

дата-время отправления запроса на выполнение,

дата-время подтверждения получения запроса (начало обработки операции),

дата-время подтверждения выполнения (окончание обработки операции), 

пользователь,

мобильное устройство,

признак завершения  выполнения операции:

черная галочка – выполнено успешно,

красная галочка – выполнено с ошибкой,

тип ошибки для неудачно завершенных операций.

 

Для работы с операциями транспортировки используются кнопки:

Создать новую операцию транспортировки

Изменить данные об операции: приоритет, пользователь, мобильное устройство;

зарегистрировать выполнение операции с выбором текущей ячейки, удачное или с ошибкой

Зарегистрировать полное удачное выполнение операции

(контейнер помещен в ячейку-приемник)

Удалить операцию

(только для невыполненных и не находящихся в обработке)

Сформировать печатные документы:

·         лист заданий на операции транспортировки

Дополнительно:

·         Повысить / Понизить приоритет операции

·         Отметить дату-время начала выполнения задания

·         "Массовое" (последовательное) подтверждение выполнения операций
