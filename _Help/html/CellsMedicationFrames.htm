Исправление состояния ячейки, предназначенной для контейнеров

Исправление состояния ячейки используется в случаях:

по данным системы в ячейке находится один контейнер, реально – другой контейнер,

по данным системы в ячейке находится один или несколько контейнеров, реально – ячейка пуста,

по данным системы ячейка пуста, реально – в ячейке находится контейнер один или несколько контейнеров.

 

Режим исправления состояния ячейки позволяет зарегистрировать "прибытие" в ячейку или "убытие" из ячейки контейнера целиком, вместе со всем товаром, который по данным системы находится в контейнере.

Режим исправления состояния ячейки, предназначенной для контейнеров, не используется для изменения товара, находящегося в ячейке (в контейнере), его количества и срока годности; для этого предназначен соответствующий режим "исправление состояния контейнера".

 

В экранной форме, предназначенной для исправления состоянии ячейки, представлены данные:

адрес и код ячейки,

ограничение по максимальному числу поддонов (самой ячейки или зоны),

фиксированные закрепления: владелец, состояние, товар/упаковка,

список контейнеров / товаров, зарегистрированных в ячейке:

код контейнера,

состояние товара,

название товара и вложение (количество штук в коробке),

количество товара в ячейке:

штук, коробок, паллет,

срок годности.

 

При исправлении состояния ячейки используются кнопки:

"Поместить" контейнер в ячейку

"Удалить" контейнер из ячейки

Отказаться от изменений при выборе контейнера

Сохранить выбор контейнера

Сохранить состояние ячейки

 

Для выбора добавляемого контейнера следует набрать его код (4 символа с ведущими нулями) либо выбрать его штрих-код в списке.

Если выбранный контейнер учитывается в какой-либо ячейке, следует подтвердить его "перемещение" из той ячейки в текущую, тогда при сохранении изменений товар этого контейнера будет просто "переписан" из одной ячейки в другую.

Если выбранный контейнер не учитывается ни в какой ячейке, его товар будет "взят" из виртуальной ячейки "Lost&Found" и помещен в текущую ячейку.

Добавленные контейнеры показаны в таблице на зеленом фоне, удаленные – на красном фоне.

 

При нажатии кнопки с изображением желтой дискеты, после подтверждения пользователя, в системе происходит "перерегистрация" товара в выбранных контейнерах в текущей ячейке; при этом создаются записи в "истории" ячеек.
