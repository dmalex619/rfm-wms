Редактирование / выполнение операции транспортировки

 

С помощью этого режима можно:

указать / изменить существующие параметры задания на транспортировку:

приоритет,

пользователь,

устройство,

зарегистрировать выполнение операции, удачное или неудачное с ошибкой.

 

Для текущей операции показаны:

контейнер:

номер,

штрих-код, 

владелец,

состояние товара,

признак разного товара,

ячейка-источник:

складская зона,

тип складской зоны,

адрес,

адрес буферной ячейки,

ячейка-приемник:

складская зона,

тип складской зоны,

адрес,

адрес буферной ячейки,

пользователь,

мобильное устройство,

приоритет,

вариант завершения операции:

контейнер находится в ячейке-источнике (местоположение контейнера не меняется),

контейнер устанавливается в буферную ячейку источника,

контейнер устанавливается в буферную ячейку приемника,

контейнер устанавливается в ячейку-приемник,

операция завершается с ошибкой (контейнер остается в текущей ячейке-источнике):

выбор типа ошибки,

флажок для блокирования ячейки-приемника.

Если требуется только изменить общие параметры задания (приоритет, пользователь, устройство), не меняя текущее расположения контейнера, следует выбрать вариант "контейнер находится в ячейке-источнике", при этом операция остается незавершенной.

При регистрации удачного выполнения операции:

если контейнер установлен в ячейку-приемник – данная операция получает статус "выполнено";

если контейнер установлен в одну из буферных ячеек (в буфер при ячейке-источнике или в буфер при ячейке-приемнике) – операция разбивается на две операции (на 2 этапа):

первый этап (текущая операция):

перемещение контейнера от ячейки-источника до выбранного буфера – получает статус "выполнено",

второй этап (создана новая операция):

перемещение контейнера от выбранного буфера до ячейки-приемника – ожидает выполнения,

При регистрации неудачного выполнения операции:

выбирается вариант "задание не может быть выполнено" и указывается тип ошибки (ячейка занята, ячейка не подходит по габаритам, …). В результате операция получает статус "выполнено с ошибкой", контейнер остается в текущей ячейке-источнике; если был установлен флажок "блокировать ячейку-приемник", ячейка-приемник получает статус "блокировано" и не может участвовать в подборе мест до тех пор, пока не будет разблокирована.  

           

 

Массовое подтверждение операций транспортировки

 

Для массового (последовательного) подтверждения выполнения операций используется та же форма, что и для редактирования / подтверждения.

Пользователь выбирает контейнер: вводит 4-хзначный код с ведущими нулями либо полный штрих-код, (можно отсканировать либо ввести вручную) и выбирает вариант завершения операции (по умолчанию – полное удачное выполнение), нажимает кнопку с изображением дискеты, получает подтверждение о выполнении операции, выбирает следующий контейнер и так далее.
