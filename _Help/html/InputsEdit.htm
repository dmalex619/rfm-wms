Редактирование прихода

 

Редактирование прихода представляет собой операцию, в ходе которой вводятся данные о фактически поступившем товаре, контейнерах и таре.

Редактировать можно только неподтвержденные приходы.

Как только сотрудник начал процесс редактирования, в приходе отметка о времени начала работы. Такой приход "блокирован" для других пользователей.

 

Экранная форма для редактирования приход состоит из двух страниц: "Атрибуты" и "Контейнеры".

 

На первой странице "Атрибуты" указаны данные:

уникальный код прихода,

дата прихода,

тип прихода,

поставщик,

владелец,

состояние товара,

примечание,

складская зона и ячейка приемки,

данные о таре:

тип тары,

количество фактически поступивших единиц тары,

данные об ожидаемом и фактически поступившем товаре (суммарное количество без учета состояния):

название товара и количество единиц товара в коробке,

количество заказанного товара ("Заказ …"):

штук / кг, коробок, паллет,

количество фактически поступившего товара ("Пришло …"), сумма по всем состояниям:

штук / кг, коробок, паллет.

Поля и таблицы тары и товаров заполнены в соответствии с ожидаемым приходом.

Большая часть полей  недоступна для изменения.

Доступны для изменения только: примечание, складская зона и ячейка приемки и данные о таре (количество поступивших единиц тары каждого типа вносится непосредственно в ячейки таблицы).

 

На второй странице "Список товаров" представлена таблица контейнеров в приходе и блок полей для оформления очередного контейнера.

При оформлении очередного контейнера вводятся данные:

состояние товара,

номер контейнера: 4 цифры с ведущими нулями (например, 0012 для контейнера с уникальным кодом 12),

для состояния "годен" может быть выбран только "свободный" контейнер,

для другого состояния может быть как "свободный" контейнер, так и другой контейнер с тем же состоянием, оформлявшийся в этом же приходе;

товар и упаковка (число штук / кг в коробке): выбор из ожидаемых в данном приходе товаров либо добавление другого существующего товара,

число коробок на паллете,

число коробок в ряду,

тип поддона (евро, финский),

высота контейнера,

фактически поступившее количество товара:

для штучного товара: число целых коробок + число штук в нецелой коробке,

для весового товара: суммарный вес в кг,

срок годности, до даты.

Введенные данные о контейнере сохраняются в буфере при нажатии на кнопку с изображением синей молнии.

Данные об обработанных контейнерах показаны в таблице:

уникальный номер контейнера,

название товара,

количество единиц товара в коробке,

количество коробок на паллете,

состояние товара,

фактическое количество товара в контейнере:

штук / кг, коробок,

срок годности (до даты),

высота контейнера (м).

Полное сохранение данных из буфера происходит при нажатии на кнопку с изображением дискеты.

 

Для работы с приходом используются кнопки:

Ввести данные по очередному контейнеру

Редактировать данные по текущему контейнеру в таблице

(изменить товар/упаковку, фактическое количество, срок годности, тип поддона и высоту контейнера)

Удалить данные о текущем контейнере в таблице

Выбрать другой товар/упаковку, если он отсутствует в списке заказанных товаров

Сохранить данные о контейнере в буфере

Отказаться от изменений в данных о контейнере

Сохранить все данные о приходе, включая сведения о контейнерах в буфере

 

Редактирование прихода может производиться в несколько этапов:

сотрудник вводит данных о нескольких контейнерах и сохраняет введенные данные нажатием кнопки с дискетой, а через некоторое время может продолжить процесс редактирования данного прихода, т.е. вводить данные об оставшихся контейнерах.

Также может быть выполнено редактирование прихода, обработка которого раньше выполнялась на другом устройстве (например, для исправления ошибок, допущенных при обработке прихода с мобильного устройства).

 

Режим редактирования прихода не предназначен для изменения заказанного (ожидаемого) количества товаров.

Все "добавленные" к приходу товары, а также все товары в состоянии, отличном от ожидаемого состояния, будут иметь заказанное (ожидаемое) количество, равное 0.

 

 

Создание нового прихода

 

В случае, если оформление ожидаемых приходов производится только в учетной системе, режим недоступен.

 

Создание нового прихода аналогично редактированию существующего прихода, с учетом следующих особенностей:

на странице "Атрибуты":

так как ожидаемый приход отсутствует, следует ввести атрибуты прихода (изначально заполняются значениями "по умолчанию"):

дата прихода,

тип прихода,

поставщик,

владелец,

состояние товара,

на странице "Контейнеры":

при оформлении контейнеров следует каждый раз выбирать новый товар/упаковку.

В созданном приходе ожидаемое количество для всех товаров равно 0.
