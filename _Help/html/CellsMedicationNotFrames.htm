Исправление состояния ячейки, предназначенной для товара (не для контейнеров)

Исправление состояния ячейки используется в случаях:

по данным системы в ячейке находится какой-либо товар в определенном количестве, с определенным сроком годности, реально – другой товар, в другом количестве, с другим сроком годности.

по данным системы в ячейке находится товар, реально – ячейка пуста,

по данным системы ячейка пуста, реально – в ячейке находится товар.

 

В экранной форме, предназначенной для исправления состоянии ячейки, представлены данные:

адрес и код ячейки,

фиксированные закрепления: владелец, состояние, товар/упаковка,

список контейнеров / товаров, зарегистрированных в ячейке:

название товара и вложение (количество штук в коробке),

количество товара в ячейке:

штук, коробок, паллет,

срок годности.

 

При исправлении состояния ячейки используются кнопки:

"Поместить" товар в ячейку

"Изменить" количество / срок годности товара в ячейке

"Удалить" товар из ячейки

Отказаться от изменений при добавлении / изменении товара

Сохранить изменения при добавлении / изменении товара

Выбрать товар/упаковку (в специальной форме)

Сохранить состояние ячейки

 

При добавлении товара в ячейку следует:

выбрать товар (в специальной форме),

указать находящееся в ячейке количество товара:

количество целых коробок + количество штук в нецелых коробках либо количество кг для весового товара, 

указать срок годности товара. 

При исправлении следует указать реально находящееся в ячейке количество (коробки + штуки либо кг) и срок годности текущего товара. 

Добавляемый в текущую ячейку товар будет "взят" из виртуальной ячейки "Lost&Found" и помещен в текущую ячейку,

"удаляемый" из текущей ячейки товар будет помещен в виртуальную ячейку "Lost&Found".

Добавленный товар показан в таблице на зеленом фоне, удаленный товар – на красном фоне, товар с измененным количеством и/или сроком годности – на желтом фоне.

 

При нажатии кнопки с изображением желтой дискеты, после подтверждения пользователя, в системе происходит "перерегистрация" товара в текущей ячейке; при этом создаются записи в "истории" ячеек.
